# 目前已经实现的功能

更新时间：2026-07-03

## A股投研报告 MVP

- 新增 `POST /api/stock-reports` SSE 入口，支持 `ticker`、`thread_id`、`report_period`、`search_mode`。
- 股票代码解析支持 `6xxxxx -> .SH`，`0xxxxx / 2xxxxx / 3xxxxx -> .SZ`，以及已带 `.SH/.SZ` 的输入。
- 新增金融数据模型：`StockSubject`、`FinancialEvidenceItem`、`FinancialSnapshot`、`FinancialMetricResult`。
- 新增 `FinancialDataProvider` 扩展点，已实现 `UploadedReportProvider` 和 `PublicMarketDataProvider`。
- 公开数据失败时不会阻断流程，会写入 `DATA_MISSING` 证据。
- 新增 `FinancialMetricEngine`，用 Java `BigDecimal` 计算营收同比、毛利率、净利率、ROE、资产负债率、经营现金流 / 净利润。
- 缺少分子或分母时返回 `MISSING_INPUT`，报告中显示“数据缺失”。
- 新增独立金融工作流：`StockResolve -> DataSnapshot -> MetricEngine -> EvidenceCollect -> InvestmentWriter -> CitationReviewer`。
- 报告固定八章节，并在底部输出“引用与数据快照”和指标公式。
- 新增 `CitationReviewer`，检查有效证据数量、报告期混用、指标证据引用和指标值展示。
- 报告统一标注“仅作研究辅助，不构成投资建议”。

## 后续优化已落地的基础能力

- FinSight 风格的轻量分阶段金融链路已拆出，不污染通用调研链路。
- 前端 Run 工作区新增“股票代码分析”模式、股票代码输入、报告期选择、金融执行轨迹。
- 前端新增财务指标卡、证据数量、缺失项数量展示。
- 新增 Bad Case 反馈接口和前端按钮：数字错、引用错、逻辑错、信息过期。
- 新增股票报告回放接口和前端回放面板，返回 snapshot + evidence + metric。
- 新增 `backend/src/main/resources/financial-eval-set.json`，包含 10 个 A 股评测样例和 6 个评测指标名称。
