## P1 待解决

### 数据源：TuShare 联调与硬化

当前状态：已新增 10 个评测样例股票的本地主档，并新增 `TushareMarketDataProvider`，可通过 `TUSHARE_API_KEY` 采集 TuShare 财务和日行情基础指标。财报重复版本、上年同期错配、期末权益冒充平均权益的问题已完成代码修正，并随 2026-07-14 后端全量 105 个测试完成验证；缓存、限速和超时硬化仍未完成。

建议方案：用真实 TuShare token 继续复核贵州茅台营收同比 6.54% 与 ROE 10.57%；随后补 provider 级缓存、限速、接口权限错误提示和更多行情序列指标。

### 网页证据：Tavily Extract 联调与质量基线

当前状态：已写入 `TavilyExtractClient`、候选 URL 优先级、批量正文提取、合格摘要降级和公开网页质量门控；配置为最多 3 个 URL、单次 10 秒、网络错误或 5xx 最多重试 1 次。相关单元测试已随 2026-07-15 后端 128 项全量回归通过，但尚未运行真实 Tavily 调用，不能确认不同财经站点的提取成功率、额外 P95 延迟和 credits 消耗。

建议方案：使用 `600519` 混合检索记录 Search/Extract 的成功 URL、失败 URL、耗时和最终有效新闻数量；至少覆盖交易所公告、普通财经正文、登录页和行情导航页，再决定是否增加域名白名单或 advanced extraction。

### 评测：真实 Golden Set 与模型基线

当前状态：离线评测 v1 已实现 20 个冻结报告样例、24 个检索标注、稳定 `chunkId`、Recall/Precision/MRR/NDCG、JSON/Markdown 产物、显式基线更新和 PR 强门禁；Judge 与真实股票/ETF 冒烟有独立手动/定时工作流，缺 Key 时明确 `SKIPPED`。当前冻结样例主要用于机制回归，尚不能代表真实线上准确率；Judge、P95 延迟和 Token 也还没有积累可审查的非零历史基线。

建议方案：从已确认的 Bad Case 中脱敏沉淀跨行业、跨报告期 Golden Set；连续运行 Judge/真实数据工作流后再提交模型质量、P95 和 Token 基线，并在数据量足够时评估趋势存储与管理页面。

### Reviewer：数字全量抽取校验

当前状态：`CitationReviewer` v3 已要求正文指标所在句引用对应原始证据，并拦截无效证据编号、阶段性 ROE 口径错误、静态 PE 无依据和跨网页行情拼接；仍没有完整抽取报告所有金额、价格、倍数和百分比并逐项比对。

建议方案：增加数字抽取器，按 `FinancialSnapshot` 与 `FinancialMetricResult` 做 0.5% 相对误差或 0.01 绝对误差校验。

### LLM：应用内端到端回归与延迟基线

当前状态：真实 `API_KEY`、DashScope 兼容地址和 `qwen3.7-max` 已完成延迟联调，压缩后真实请求为 50.859 秒。Writer 增加最多 18 条紧凑有效证据索引和正文就近引用要求后，相关代码已随 2026-07-15 后端 128 项全量回归通过；真实应用仍需重启并重新建立延迟与引用质量基线。

建议方案：重启后端后用 `600519` 混合检索连续生成报告，确认 `generation-mode: llm`、正文关键指标带对应 `[E#]`、一季度 ROE 标注未年化、低质量网页不进入正文，同时比较新旧 Writer 的 P50/P95；若降级，数据库应记录 `DEGRADED` 和稳定原因分类。

### 前端：独立股票报告页

当前状态：股票报告页目前集成在 Run 工作区，通用研究模式及失效 `/api/chat` 调用已经删除，页面只保留 A股/ETF 证券代码入口，并已有指标卡、风险评分、合规状态、数据源耗时、证据分类、反馈和回放；2026-07-15 前端生产构建通过，但还不是独立股票报告页。

建议方案：抽出独立股票报告页，增加证据列表详情、报告时间线筛选、风险维度筛选和多版本对比。

### 资产类型：ETF 深度数据

当前状态：已新增 `StockAssetType.ETF`，支持沪市 `5xxxxx` 和深市 `15xxxx/16xxxx/18xxxx` ETF 解析；ETF 不套普通股票财务指标，TuShare 优先走 `fund_daily`，报告模板改为 ETF 口径。基金净值、规模、持仓、跟踪误差和申赎清单还未完整接入。

建议方案：后续接入基金净值、规模、持仓、跟踪误差、申赎清单和跟踪指数信息，并按 token 权限增加 provider 级缓存、限速和接口权限错误提示。

### 可视化：ECharts 行情图表

当前状态：外部项目有 K 线、MACD、RSI 图表，本项目已接入 TuShare `daily_basic` 基础估值/市值证据，尚未落地日线序列和 ECharts 前端图表组件。

建议方案：后续接入 TuShare `daily`/`weekly` 行情序列后，用 ECharts 在前端展示价格、成交量、均线、RSI，不把图表生成放到 Python 脚本。

### 多 Agent：多空研究员辩论

当前状态：外部项目有多空研究员辩论和风险经理角色，本项目目前先做确定性风险评分与合规审查（`FinancialRiskScorer`、`FinancialComplianceReviewer` 和结构化报告分区）。

建议方案：新增 `BullBearDebateService`，要求每个论点绑定证据 ID，并由合规审查过滤荐股、仓位、保证收益表达。

### 执行资源：并发受控与观测

当前状态：已把股票报告和 provider fan-out 改为 Spring 管理的有界执行器；股票任务具备数据库心跳、最多 3 次恢复和 Micrometer 指标。仍缺任务主动取消、provider 单项超时和线程池队列指标。

建议方案：后续增加任务取消、provider 单项超时和执行队列观测指标。
