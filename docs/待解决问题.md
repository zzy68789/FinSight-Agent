# 待解决问题

| 优先级 | 类型 | 待解决问题 | 当前状态 | 建议方案 |
| --- | --- | --- | --- | --- |
| P1 | 数据源 | 行情、估值和新闻仍依赖搜索摘要或缺失标记，缺少可授权实时 A 股数据源。 | 已新增 10 个评测样例股票的本地主档，股票解析能补全公司名称/行业，并输出 `LOCAL_CONTEXT` 证据。 | 接入可授权的 A 股行情、估值和新闻数据源，继续走 `FinancialDataProvider`，不把外部 SDK 泄漏到工作流。 |
| P1 | 评测 | 自动评测已能加载默认样例并输出指标，但还没有独立命令行批量评测报告。 | 已新增 `FinancialEvaluationService`、`evaluation` SSE 事件和 Maven 单测，覆盖 claim/citation/numeric/keypoint 指标。 | 增加 CLI/测试 runner 批量跑 10 个样例，输出 JSON/Markdown 评测报告，并把阈值作为 CI 门控。 |
| P1 | Reviewer | `CitationReviewer` 已检查证据数量、公式引用、报告期和指标展示值，但还没有完整抽取报告所有数字并逐项比对。 | MVP 规则已覆盖核心链路。 | 增加数字抽取器，按 `FinancialSnapshot` 与 `FinancialMetricResult` 做 0.5% 相对误差或 0.01 绝对误差校验。 |
| P1 | 前端 | 股票报告页目前集成在 Run 工作区，还不是独立股票报告页。 | 已有股票模式、指标卡、风险评分、合规状态、数据源耗时、证据分类、反馈、回放。 | 抽出独立股票报告页，增加证据列表详情、报告时间线筛选、风险维度筛选和多版本对比。 |
| P1 | 可视化 | 外部项目有 K 线、MACD、RSI 图表，但当前还没有稳定 A 股行情序列和前端图表组件。 | 已先落地风险评分、指标和证据面板，没有引入 Python/yfinance/matplotlib。 | 接入可授权行情序列后，用 ECharts 在前端展示价格、成交量、均线、RSI，不把图表生成放到 Python 脚本。 |
| P1 | 多 Agent | 外部项目有多空研究员辩论和风险经理角色，本项目目前先做确定性风险评分与合规审查。 | 已有 `FinancialRiskScoringService`、`FinancialComplianceReviewer` 和结构化报告分区。 | 新增 `BullBearDebateService`，要求每个论点绑定证据 ID，并由合规审查过滤荐股、仓位、保证收益表达。 |
| P1 | 执行资源 | 长流程任务和金融 provider 并发需要受控线程池和超时策略。 | 已把通用研究、股票报告和 provider fan-out 改为 Spring 管理的有界执行器。 | 后续增加任务取消、provider 单项超时和执行队列观测指标。 |
| P2 | 产品化 | 尚未实现行业横向对比、同行估值区间、组合监控、价格/新闻/风险预警。 | 不属于 MVP，当前明确不做荐股、交易或回测。 | 在数据源稳定后，以只读研究辅助为边界逐步增加 ECharts 图表和订阅式提醒。 |
| P2 | 运维 | 已有升级 SQL，但没有自动迁移工具。 | 新库由 `schema.sql` 建表，旧库需手动执行 `upgrade-financial-report.sql`。 | 引入 Flyway 或 Liquibase，并把历史升级脚本纳入版本化迁移。 |
