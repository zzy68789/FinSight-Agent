# 外部金融项目借鉴执行计划

更新时间：2026-07-03

## 一句话结论

`wudengyao/stock-analysis-team` 值得借鉴的是“多角色金融团队、风险评分维度、HTML/图表化报告呈现”；`bcefghj/multi-agent-financial-report` 值得借鉴的是“DataCollector + Sentiment 并行 fan-out、Market/Writer/Compliance 串行 pipeline、合规 issue 列表”。本项目不照搬 Python/yfinance、美股 SEC 数据源、自动交易和仓位建议，而是把这些设计压力落到现有 Java/Spring Boot/LangGraph4j/Vue 的 A 股研究辅助链路中。

## 可借鉴设计清单

| 优先级 | 外部设计思路 | 本项目落地点 | 建议改造方式 | 简历/面试价值 | 风险边界 |
| --- | --- | --- | --- | --- | --- |
| P0 | Provider fan-out 并行 | `FinancialSnapshotBuilder` | 用 `CompletableFuture` 并行调用上传报告 provider 和公开数据 provider，并输出每个 provider 的耗时/状态。 | 能讲清楚多 Agent/多数据源的并行编排与长任务观测。 | 不承诺固定延迟优化比例。 |
| P0 | 风险评分维度 | `FinancialRiskScoringService`、前端股票质检卡 | 参考五维权重：基本面30%、技术面25%、情绪20%、消息15%、市场环境10%，缺数据显式给中性偏谨慎分。 | 能体现“金融不是只写报告，还做风险约束”。 | 不输出仓位建议、买卖点或交易执行。 |
| P0 | 合规 issue 列表 | `FinancialComplianceReviewer`、`CitationReviewer` | 检查免责声明、保证收益、内幕信息、证据章节、有效证据数量，返回 issues 和 score。 | 金融 AI 项目必须有合规闭环，面试可解释规则与责任边界。 | 本地规则审查，不冒充监管合规系统。 |
| P1 | 多角色报告分栏 | `InvestmentReportWriter`、`App.vue` | 报告和前端拆成财务指标、风险评分、合规审查、证据账本几个可扫描区域。 | 前端产品化更像投研工作台。 | 不做营销式大屏，保持工作台密度。 |
| P1 | 技术指标可视化 | 后续 `TechnicalIndicatorService` + ECharts | 接 A 股行情后计算 MA、MACD、RSI，并前端图表化。 | 增强金融垂直能力。 | 当前缺稳定行情数据源，暂不作为 P0。 |
| P2 | 多空研究员辩论 | 后续 `BullBearDebateService` | 基于证据和风险项生成多空论点，不给交易指令。 | 可作为多 Agent 亮点。 | 容易走向荐股，必须加免责声明与证据约束。 |

## 本轮自动执行范围

1. 后端：新增 provider 并行采集元数据、风险评分服务、合规审查服务。
2. 后端：股票报告 SSE 增加 `risk_assessment` 与增强 `reviewer` payload。
3. 前端：股票模式增加风险评分、合规问题、provider 耗时、证据分类显示。
4. 文档：更新 `PROBLEM_LOG.md`、`待解决问题.md`、`目前已经实现的功能.md`。
