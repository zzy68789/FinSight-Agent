## P1：数据源硬化

### TuShare Provider 真实联调

当前状态：`TushareMarketDataProvider` 已接入 `income`、`balancesheet`、`cashflow`、`daily_basic` 和 ETF `fund_daily` 映射；财报版本去重、严格同期匹配、年初/期末权益 ROE 口径及基础语义校验已随 2026-07-14 后端全量 105 个测试完成验证；未配置 token 时不访问外部接口。

待实现：

- 使用真实 TuShare token 联调普通股票和 ETF 样例。
- 补充 provider 级缓存，避免同一任务/同一报告期重复请求。
- 增加接口限速、权限不足、额度不足、数据为空的分类错误提示。
- 把 `finsight.market.tushare.timeout` 真正用于连接和读取超时，防止外部 HTTP 挂起占满 `financialProviderExecutor`。

验收标准：

- 外部接口超时会在配置时间内失败，并写入 `DATA_MISSING` 证据。
- provider stage 能展示失败类型、耗时和证据数量。

## P1/P2：评测与 Reviewer

### 独立批量评测 Runner

当前状态：已完成首版。`FinancialEvaluator` v4 对所有证券执行线上硬门禁并把来源可用性降为告警；`FinancialEvaluationRunner` 使用 12 个股票/ETF 冻结正例、8 个规则负例和 24 个检索问题，输出 JSON/Markdown，执行 Recall@K、Precision@K、MRR、NDCG@K、数值误差和基线回归。PR GitHub Actions 已接入确定性强门禁，Judge/真实数据工作流保持非阻断。

验证：2026-07-15 执行 `mvn.cmd "-Dtest=FinancialEvaluationRegressionTest" test` 通过；随后后端全量 128 项测试通过，Judge 和真实数据冒烟因默认关闭跳过 2 项。

剩余可增强：

- 把已确认的生产 Bad Case 脱敏后持续加入 `dataset-v1`，扩大行业、报告期和异常数据覆盖。
- 在积累多次 Judge/真实数据结果后建立非零模型质量、P95 延迟和 Token 基线。
- 后续再建设数据库趋势存储和评测管理页面；首版不增加线上 API 或表结构。

### 报告数字全量抽取与误差校验

当前状态：`CitationReviewer` v3 已把检查范围从完整附录收紧到正文，指标值所在句必须引用对应原始字段，并拦截无效编号、阶段性 ROE 口径错误、静态 PE 无依据和跨网页行情拼接；仍未实现全文所有金额、倍数、价格和百分比的通用数字抽取。

待实现：

- 新增数字抽取器，识别百分比、金额、市值、倍数、分数等表达。
- 与 `FinancialSnapshot` / `FinancialMetricResult` 做 0.5% 相对误差或 0.01 绝对误差校验。
- 对无法追溯到证据的关键数字标记 reviewer issue。

验收标准：

- LLM 或模板写错关键数字时，reviewer 能 fail-closed。
- 报告中每个关键财务数字都有证据或指标结果来源。
