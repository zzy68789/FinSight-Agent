## P0：重构收口

### 前端通用研究模式清理

当前状态：后端通用 deep-research 链路已经移除，`ChatController`、`ResearchGraphFactory`、通用 `agent/node`、通用 `ResearchTaskService` 等文件不再存在；但前端仍保留“通用研究”按钮、`streamChat('/chat')` 调用和部分 FinSight 通用研究文案。

待实现：

- 移除或隐藏前端“通用研究”模式。
- 删除 `frontend/src/services/api.js` 中失效的 `streamChat`，或改造成金融报告专用入口。
- 将首页、登录页、任务区、按钮和提示文案统一为 FinSight / 金融投研 Agent 定位。
- `StatusFlow` 默认聚焦股票报告阶段，不再展示 Planner/Researcher/Writer/Reviewer 通用链路作为主流程。

验收标准：

- 前端运行时不会请求已删除的 `/api/chat`。
- 用户进入 Run 工作区后默认看到证券代码分析路径。
- `npm.cmd run build` 通过。

### 项目标识统一

当前状态：已完成。Java 根包、Spring 应用名、Maven 坐标、配置命名空间、MySQL、Redis key、Chroma collection、前端存储键、README 和关键页面文案均统一为 FinSight；只有三类 API Key 使用 `API_KEY`、`TAVILY_API_KEY`、`TUSHARE_API_KEY` 环境变量，其余配置保持简洁直写。

验证：2026-07-11 后端 91 个测试通过；真实 MySQL `finsight` 空库完成 Flyway V1→V2；前端生产构建通过。前端通用研究入口清理仍由上一节单独跟踪，不与品牌标识重命名混为一项。

## P1：数据源硬化

### TuShare Provider 真实联调

当前状态：`TushareMarketDataProvider` 已接入 `income`、`balancesheet`、`cashflow`、`daily_basic` 和 ETF `fund_daily` 映射；未配置 token 时不访问外部接口。

待实现：

- 使用真实 TuShare token 联调普通股票和 ETF 样例。
- 补充 provider 级缓存，避免同一任务/同一报告期重复请求。
- 增加接口限速、权限不足、额度不足、数据为空的分类错误提示。
- 把 `finsight.market.tushare.timeout` 真正用于连接和读取超时，防止外部 HTTP 挂起占满 `financialProviderExecutor`。

验收标准：

- 外部接口超时会在配置时间内失败，并写入 `DATA_MISSING` 证据。
- provider stage 能展示失败类型、耗时和证据数量。
- 有针对 TuShare 超时和错误响应的单元测试。

### 多数据源质量打分

当前状态：证据已有 source、metricName、confidence、missingReason 等字段，但不同来源之间的质量权重仍较粗。

待实现：

- 为上传报告、本地主档、TuShare、Tavily fallback 建立来源权重。
- 对新闻/搜索结果增加时效性、来源可信度和重复度评分。
- 在报告“引用与数据快照”中展示关键证据来源优先级。

验收标准：

- 同一指标多来源冲突时，报告能说明采用哪个来源及原因。

## P1：评测与 Reviewer

### 独立批量评测 CLI

当前状态：`FinancialEvaluationService` 已接入默认样例，并在命中样例时通过 SSE 输出 evaluation 结果。

待实现：

- 增加命令行或测试 runner，批量跑 `financial-eval-set.json` 的 10 个样例。
- 输出 JSON 和 Markdown 评测报告。
- 支持设置 claim/citation/numeric/keypoint 阈值，作为 CI 门控预留。

验收标准：

- 一条命令可以批量生成评测报告。
- 失败样例能定位到缺失关键点、引用不足、数字不一致或合规表达问题。

### 报告数字全量抽取与误差校验

当前状态：`CitationReviewer` 已检查指标证据引用和部分指标值展示，但还没有抽取报告全文所有数字进行逐项比对。

待实现：

- 新增数字抽取器，识别百分比、金额、市值、倍数、分数等表达。
- 与 `FinancialSnapshot` / `FinancialMetricResult` 做 0.5% 相对误差或 0.01 绝对误差校验。
- 对无法追溯到证据的关键数字标记 reviewer issue。

验收标准：

- LLM 或模板写错关键数字时，reviewer 能 fail-closed。
- 报告中每个关键财务数字都有证据或指标结果来源。

当前状态：ETF 已支持代码解析和 `fund_daily` 基础行情，报告模板已切换为基金口径；基金净值、规模、持仓、跟踪误差、申赎清单和跟踪指数仍未完整接入。

### 多空研究员辩论

当前状态：当前链路先做确定性风险评分与合规审查，没有独立的多空辩论角色。

待实现：

- 新增 `BullBearDebateService`。
