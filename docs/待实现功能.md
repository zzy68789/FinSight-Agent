## P0：后端基础能力待完善

### MySQL 持久化与一致性

已完成：无 H2、默认 MySQL、`schema.sql` MySQL 方言、`JdbcClient` 写入四张核心表、`ResearchTaskRepository.create()` JDBC 主键返回。

待实现：

- 增加 Repository 级 MySQL 集成测试（当前只有 Controller 层测试，缺 Repository 集成测试）。
- 视情况增加事务，避免 Agent 执行中途失败时数据库状态不一致。
- 增加更完整的查询条件，例如创建时间范围。

验收标准：

- 执行一次调研后，`research_task`、`agent_step_log`、`report`、`checkpoint` 均有数据。
- 任务失败时 `research_task.status` 能正确变为 `FAILED`。

## P1：大模型应用能力增强

### RAG / Hybrid 检索增强

已完成：BM25 + ChromaDB 向量混合检索、document/hybrid/web 三策略、相关性阈值、LOCAL/WEB evidence 标记、本地向量 fallback。

待实现：

- 增加 query rewrite 或 query expansion。
- 增加 rerank 层。
- 增加更正式的 evidence schema，而不是仅用 Markdown 文本拼接。
- 增加检索质量评估数据集。

验收标准：

- 能说明每条结论来自本地文档还是 Web 搜索。
- document 模式无相关文档时不编造。
- hybrid 模式本地证据不足时能自动补充 Web 结果。

### LLM / LangChain4j 使用增强

已完成：`LangChain4jLlmClient`、`RetryingLlmClient` 失败重试 fallback、`PlanParser`、`ReviewResultParser` fail-closed、fast/smart 双模型、`DRAI_LLM_*` 环境变量隔离。

待实现：

- 增加 traceId / taskId 维度的模型调用审计日志。
- 增加工具调用能力，例如搜索工具、文件工具。
- 增加更严格的 JSON schema 约束和修复重试。

验收标准：

- Planner、Reviewer 的结构化输出稳定。
- 模型调用失败时 Agent 不会静默生成错误内容。
- 日志中能追踪每次模型调用的耗时和失败原因。

### 联网搜索能力增强

已完成：`SearchSource` 抽象、`TavilySearchSource`、`EnhancedSearchService`（重试/去重/质量过滤/fallback）、evidence 展示 source/title/URL。

待实现：

- 扩展多搜索源：Brave Search、Jina、arXiv、GitHub Search。
- 搜索结果持久化到 MySQL，便于报告溯源和历史查询。
- 搜索源质量打分和权重配置。

## P2：Agent 工作流增强

已完成：LangGraph4j 基础工作流、Reviewer 失败回到 Planner、Refiner 基于历史报告修改、Planner 子任务运行态、Researcher 按子任务检索、Reviewer 无证据 fail-closed。

待实现：

- 增加 Supervisor 节点，统一调度任务。
- 增加 Clarifier 节点，在用户问题不清晰时先澄清。
- 增加真正并行 Researcher，分别处理多个子问题。
- 增加更细粒度 evidence checker，检查报告每条结论是否有证据支撑。
- 增加最大执行轮数、最大搜索次数、最大 token 成本控制。
- 增加人工反馈 HITL 流程。

## P3：金融垂直能力增强

对应 `docs/外部金融项目借鉴执行计划.md` 中标记为“待后续”的项。

待实现：

- ETF 深度数据：基金净值、规模、持仓、跟踪误差、申赎清单、跟踪指数。
- 行情序列可视化：接入 TuShare `daily`/`weekly` 后用 ECharts 展示价格、成交量、均线、RSI（不引入 Python）。
- 多空研究员辩论：`BullBearDebateService`，每个论点绑定证据 ID，合规审查过滤荐股/仓位/保证收益表达。
- 独立批量评测 CLI：批量跑 10 个样例，输出 JSON/Markdown 评测报告，阈值作为 CI 门控。
- Reviewer 数字抽取器：抽取报告所有数字，按 0.5% 相对误差或 0.01 绝对误差逐项校验。
- TuShare 真实 token 本地联调，补 provider 级缓存、限速、接口权限错误提示。
- provider 级任务取消、单项超时和执行队列观测指标。
- 数据库迁移工具：引入 Flyway 或 Liquibase，历史升级 SQL 纳入版本化迁移。
